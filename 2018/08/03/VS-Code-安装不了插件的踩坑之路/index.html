VS Code 安装不了插件的踩坑之路

Posted on 2018-08-03

前段时间更新了 VS Code 后，突然就安装不了插件了，因为一直忙于赛飞奇的项目，也就放着不理了，今天想练一下 react, 于是就想着装个插件，顺带解决一直以来遗留的问题。

第一个想法是卸载再安装，但是试了几次后还是不行，因为每次重新安装后，VS Code 的之前配置文件还是没有彻底删除，于是我去查了一下，ubuntu 彻底删除应用的命令：

我试着去执行这两个命令的时候，第二个命令一直报错，怀着侥幸的心理我再次卸载重装，结果。。。还是不行

  • 谷歌了好多次，大多都是说线上安装不了，怎么在线下安装的问题，诸如梯子之类的什么。。。
  • 但是我是有用网络代理的，线下也是安装不了，所以问题应该不是代理的问题

接着终于在 VS Code 的 github 找到了相关的issue
按照他的教程

Hit Ctrl Shift P to open the command palette, type Shared Process and choose Developer: Toggle Shared Process. In the devtools which opens, switch to the Network tab and retry installing. Show us the error that shows up in the Network tab.

居然发现在 VS Code 居然能打开类似与浏览器的控制台，之前一直不知道….还是太菜了..唉

于是这给了我一些灵感，在控制台的 network 捕捉到了我发起安装插件的网络请求，报了个错

报错信息大概是这样的，具体的忘了保存了，于是我顺着报错的信息给出的路径去找那个文件，发现我是没有这个文件的权限的， 为了获取执行权限， 我用了 chmod 指令修改文件的权限

然后我估摸着重启一下，没想到居然可以了，原来一切的根源就是那个文件没有权限访问，但VS Code 主页上又不报错误信息，唉，有点毒。

记录一下，希望下次遇到类型问题时，解决问题的途径和方法能优化一丢丢。
